## 📋 项目概述

AI秘书采用 **读写执行体与大模型中枢严格解耦** 的极简架构，基于 HarmonyOS 原生 ArkTS 与 relationalStore 构建。系统支持端云双态运行：优先对接云端 GLM-5 大模型，网络异常时无缝降级至本地 Mock 优雅降级引擎，确保无网环境 100% 局部可用。

**核心数据流：**

```
┌─────────────┐     ┌─────────────────┐     ┌─────────────┐
│   用户输入   │────▶│  ReadExecutor   │────▶│  LLMService │
│  (自然语言)  │     │ 全息上下文组装   │     │ 中枢决策/双态 │
└─────────────┘     └─────────────────┘     └──────┬──────┘
                                                    │
                         ┌──────────────────────────┘
                         ▼
              ┌─────────────────────┐
              │   本地安全探针       │  ◄── 敏感词遍历 / 特征拦截
              │   SecurityChecker   │
              └─────────────────────┘
                         │
                         ▼
              ┌─────────────────────┐
              │   WriteExecutor     │  ◄── JSON 反序列 / 多态路由
              │   多态分发引擎       │
              └─────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
    ┌─────────┐   ┌─────────────┐   ┌─────────────┐
    │  Chat   │   │  Schedule   │   │  Study/Psych│
    │ 直通渲染 │   │ 弹窗确认落盘 │   │ 专属逻辑    │
    └─────────┘   └─────────────┘   └─────────────┘
```

| 角色 | 核心职责 | 主责文件 | 协作边界 |
|------|---------|---------|---------|
| **组长** | UI 重构与主逻辑中枢管控 | `Index.ets`、`ActionDialog.ets` | 统筹状态流转，定义全局组件规范 |
| **组员A** | 大模型调度策略与 Mock 灾备机制 | `LLMService.ets`、`SecurityChecker.ets` | 双态调度、Prompt 工程、安全探针 |

## 🔌 模块接口规范

### 1️⃣ 组员B：Read 感知层 (`ReadExecutor.ets`)

#### 职责
负责全息环境上下文组装，将系统状态与时间信息无缝注入 Prompt，为大模型决策提供完整语境。

#### 对外接口

```typescript
class ReadExecutor {
  /**
   * 获取全息系统信息
   * @returns SystemContext 包含时间、设备、网络等上下文
   */
  static getSystemContext(): SystemContext

  /**
   * 组装完整 Prompt
   * @param userInput 用户原始输入
   * @param history   最近 N 条对话历史（从 DatabaseManager 读取）
   * @returns 可直接送入 LLMService 的完整 Prompt 字符串
   */
  static buildPrompt(userInput: string, history: MessageRecord[]): string
}

interface SystemContext {
  currentTime: string      // 2026-04-18 13:45 格式
  date: string             // 2026/4/18 星期六
  weekday: string          // 星期六
  batteryLevel?: number    // 电量百分比（可选扩展）
  networkType?: string     // WiFi / 移动数据 / 无网络（可选扩展）
}
```

#### Prompt 组装模板

```
你是一个 AI 个人助理秘书，请严格按 JSON 格式返回结果。

【当前系统上下文】
- 时间：{{currentTime}}
- 日期：{{date}} {{weekday}}
- 设备电量：{{batteryLevel}}%
- 网络状态：{{networkType}}

【最近对话历史】
{{history}}

【用户最新输入】
{{userInput}}

【返回格式约束】
必须且仅返回如下 JSON，不要添加任何解释性文字：
{
  "action": "chat|schedule|study|psych",
  "reply": "给用户看的自然语言回复",
  "data": { "根据不同 action 填充" }
}
```

### 2️⃣ 组员A：大模型中枢 (`LLMService.ets`)

#### 职责
负责大模型 API 调度与双态灾备。优先调用云端 GLM-5；网络异常、超时或 API 失败时，自动降级至本地 Mock 优雅降级引擎。

#### 对外接口

```typescript
class LLMService {
  /**
   * 双态大模型调用入口
   * @param prompt 由 ReadExecutor 组装的完整 Prompt
   * @returns JSON 字符串，格式受严格约束
   */
  static async generateResponse(prompt: string): Promise<string>

  /**
   * 当前运行模式查询
   * @returns "cloud" | "mock"
   */
  static getCurrentMode(): string
}
```

#### 内部实现要点

**A. 云端模式（Cloud）**
```typescript
private static async cloudCall(prompt: string): Promise<string> {
  // 对接 GLM-5 API
  // POST https://api.glm-5.example/v1/chat/completions
  // Headers: Authorization: Bearer {API_KEY}
  // Body: { model: "glm-5", messages: [{role: "user", content: prompt}] }
  
  // 若请求失败（网络错误 / HTTP 非 200 / 响应超时 > 5s），自动抛异常
}
```

**B. 降级模式（Mock）**
```typescript
private static async mockCall(userInput: string): Promise<string> {
  // 纯本地特征词阻击，零网络依赖
  const keywords = {
    schedule: ['日程', '会议', '安排', '开会', '提醒'],
    study:    ['学习', '课程', 'Python', '进阶', '技能'],
    psych:    ['累', '焦虑', '好烦', '抑郁', '不开心'],
  }
  
  // 匹配优先级：schedule > study > psych > chat（默认）
  // 返回格式与云端保持 100% 一致的标准化 JSON
}
```

#### 降级触发条件
1. `navigator.connection.type === ConnectionType.NONE`
2. 云端 API 请求超时（> 5000ms）
3. 云端返回 HTTP 非 200 状态码
4. 返回 JSON 解析失败（兜底）

### 3️⃣ 组员A：本地安全探针 (`SecurityChecker.ets`)

#### 职责
内置敏感词语料库，通过高效遍历匹配，前置拦截违规注入。针对高危写操作，强制唤起 ActionDialog 进行物理级确认。

#### 对外接口

```typescript
class SecurityChecker {
  /**
   * 输入安全检测
   * @param content 用户输入或 LLM 返回内容
   * @returns isSafe 是否通过检测
   */
  static checkInput(content: string): boolean

  /**
   * 判断操作是否需要物理级确认
   * @param action 动作类型
   * @returns 是否需要弹窗
   */
  static needPhysicalConfirm(action: ActionType): boolean
}

type ActionType = 'chat' | 'schedule' | 'study' | 'psych'
```

#### 敏感词库示例（本地维护）
```typescript
private static SENSITIVE_WORDS = [
  '暴力', '恐怖', '赌博', '诈骗', // ... 按需扩展
]
```

#### 权限最小化原则
在 `module.json5` 中严格按需申请：
```json
{
  "requestPermissions": [
    { "name": "ohos.permission.INTERNET" },
    { "name": "ohos.permission.GET_NETWORK_INFO" }
  ]
}
```
网络权限与本地存储权限隔离管控，禁止过度申请。

### 4️⃣ 组员C：Write 分发引擎 (`WriteExecutor.ets`)

#### 职责
负责 JSON 精准反序列化与多态 Action 路由分发。针对 `schedule` 场景，进入纯手写 Regex 实体提取管线，极速剥离日期、时间、事件要素。

#### 对外接口

```typescript
class WriteExecutor {
  /**
   * 解析 LLM 返回的 JSON 并路由到对应处理器
   * @param llmResponse JSON 字符串
   * @returns ActionResult 结构化结果
   */
  static parseAndRoute(llmResponse: string): ActionResult
}

interface ActionResult {
  action: ActionType
  reply: string
  data?: object
  needConfirm: boolean   // 由 SecurityChecker 决定
}
```

#### 多态路由表

| Action | 路由目标 | 处理逻辑 |
|--------|---------|---------|
| `chat` | ArkUI 渲染层 | 直接渲染 reply 文本 |
| `schedule` | Regex 提取管线 + 弹窗确认 | 提取 Date/Time/Event → ActionDialog → 数据库落盘 |
| `study` | 学习推荐引擎 | 结构化技能路径生成 |
| `psych` | 心理卫士逻辑 | 匹配安抚白名单 → 专属回复 |

#### Schedule Regex 提取管线

```typescript
// 示例输入："帮我安排明天下午3点开会"
// 输出：{ date: "2026-04-19", time: "15:00", event: "开会" }

private static extractScheduleEntities(text: string): ScheduleEntity {
  const datePatterns = [/明天/, /后天/, /下周([一二三四五六日])/]
  const timePatterns = [/([上下])午(\d+)点/, /(\d+):(\d+)/]
  const eventPatterns = [/(?:安排|提醒|去)(.+)/]
  
  // 极速剥离，拒绝沉重三方库，单条解析 < 5ms
}
```

### 5️⃣ 组员B：持久化引擎 (`DatabaseManager.ets`)

#### 职责
基于 HarmonyOS relationalStore 实现严格单例模式的高性能本地持久化引擎，支撑零延迟启动与分页防爆表。

#### 对外接口

```typescript
class DatabaseManager {
  /**
   * 获取单例实例
   */
  static getInstance(): DatabaseManager

  /**
   * 初始化数据库（零延迟启动）
   */
  async init(): Promise<void>

  // ---------- Messages 表 ----------
  async saveMessage(content: string, isUser: boolean, timestamp: string): Promise<number>
  async getRecentMessages(count: number): Promise<MessageRecord[]>

  // ---------- Schedules 表 ----------
  async saveSchedule(time: string, event: string, status: string): Promise<number>
  async getAllSchedules(): Promise<ScheduleRecord[]>
  async markScheduleDone(id: number): Promise<void>

  // ---------- Users 表 ----------
  async getUserInfo(): Promise<UserRecord>
}
```

#### 单例实现

```typescript
class DatabaseManager {
  private static instance: DatabaseManager
  private db: relationalStore.RdbStore | null = null

  private constructor() {}

  public static getInstance(): DatabaseManager {
    if (!DatabaseManager.instance) {
      DatabaseManager.instance = new DatabaseManager()
    }
    return DatabaseManager.instance
  }
}
```

#### 表结构定义

```sql
-- Users 表（用户身份）
CREATE TABLE IF NOT EXISTS Users (
  user_id INTEGER PRIMARY KEY,
  username TEXT NOT NULL,
  auth_token TEXT
);

-- Messages 表（聊天记录）
CREATE TABLE IF NOT EXISTS Messages (
  msg_id INTEGER PRIMARY KEY AUTOINCREMENT,
  role TEXT NOT NULL,        -- 'user' | 'assistant'
  content TEXT NOT NULL,
  timestamp TEXT NOT NULL
);

-- Schedules 表（日程安排，分页防爆表）
CREATE TABLE IF NOT EXISTS Schedules (
  task_id INTEGER PRIMARY KEY AUTOINCREMENT,
  time TEXT NOT NULL,
  event TEXT NOT NULL,
  status TEXT DEFAULT 'pending'  -- 'pending' | 'done'
);
```

#### 分页防爆表策略
查询 Messages 时强制使用 `LIMIT` + `ORDER BY DESC`，避免大数据量下内存溢出：
```typescript
const predicates = new relationalStore.RdbPredicates('Messages')
predicates.orderByDesc('msg_id').limit(50)  // 单次最多 50 条
```

## 📡 标准化 JSON 通信契约

**LLM → WriteExecutor 数据格式（强制约束）：**

```typescript
interface LLMResponse {
  action: 'chat' | 'schedule' | 'study' | 'psych'
  reply: string        // 展示给用户的自然语言
  data?: {
    // action = schedule 时必填
    time?: string      // e.g. "明天15:00"
    event?: string     // e.g. "开会"
    
    // action = study 时可选
    topic?: string
    resources?: string[]
    
    // action = psych 时可选
    emotion?: string
    suggestion?: string
  }
}
```

**错误处理：**
若 LLM 返回非 JSON 或字段缺失，`WriteExecutor` 捕获异常后降级为 `action: "chat"`，reply 为原始文本，确保系统不崩溃。
